Chạy môi trường Development (Localhost)
Dùng hàng ngày để phát triển — hot-reload, debug, không cần Docker
📌
Cần 2 terminal song song. Backend chạy port 5244, Frontend chạy port 5173.

Khởi động SQL Server

Đảm bảo SQL Server đang chạy trên máy. Kiểm tra qua SQL Server Configuration Manager hoặc Services.

✅
Connection string mặc định trong backend/src/WorkForceManager.WebApi/appsettings.json → "Server=(localdb)\\mssqllocaldb"

Terminal 1 — Chạy Backend

Mở PowerShell/CMD tại thư mục gốc dự án:

# Chạy backend (tự động migrate DB và seed data lần đầu) dotnet run --project backend/src/WorkForceManager.WebApi # Đợi đến khi thấy dòng: Now listening on: http://localhost:5244
📖
Swagger UI: http://localhost:5244/swagger — test API trực tiếp

Terminal 2 — Chạy Frontend

Mở PowerShell/CMD mới:

# Vào thư mục frontend cd frontend # Lần đầu: cài dependencies (chỉ làm 1 lần hoặc sau khi thêm package) npm install # Khởi động dev server npm run dev # Đợi đến khi thấy: Local: http://localhost:5173/

Mở trình duyệt

Vào địa chỉ: http://localhost:5173

Đăng nhập bằng tài khoản demo ở mục Tài khoản mặc định.

Deploy Production (Docker — 1 lệnh)
Chạy toàn bộ hệ thống bằng Docker Compose — không cần cài .NET hay Node

Cài Docker Desktop (Windows) hoặc Docker Engine (Linux/VPS)

Tải tại https://www.docker.com/products/docker-desktop/

Sau khi cài xong, kiểm tra:

docker --version docker compose version

Tạo file .env tại thư mục gốc

Copy từ file mẫu và điền giá trị thực:

# Mật khẩu SA cho SQL Server (tối thiểu 8 ký tự, có chữ hoa + số + ký tự đặc biệt) SA_PASSWORD=MatKhauManhCuaBan_2024! # JWT Secret key (chuỗi ngẫu nhiên dài ≥ 32 ký tự) JWT_SECRET=SaigonSpices_JWT_SecretKey_WorkForce_Manager_2024 # URL công khai của server (IP hoặc domain) PUBLIC_URL=http://192.168.20.127 # Bật/tắt Swagger (true = bật, false = tắt khi production thật) ENABLE_SWAGGER=true # Có seed data mẫu hay không SEED_ENABLED=true
⚠️
Không commit file .env lên Git! File này chứa mật khẩu và secret key.

Build và khởi động toàn bộ

# Chạy từ thư mục gốc dự án docker compose -f docker-compose.prod.yml up -d --build # Đợi ~3-5 phút lần đầu (download images + build) # Xem log realtime: docker compose -f docker-compose.prod.yml logs -f
✅
Khi thấy Now listening on: http://[::]:8080 ở log backend là backend đã sẵn sàng.

Quản lý Docker Container
Các lệnh thường dùng để vận hành hàng ngày
▶️ Khởi động / Dừng
# Khởi động (không rebuild) docker compose -f docker-compose.prod.yml up -d # Dừng (giữ data) docker compose -f docker-compose.prod.yml down # Dừng và xóa volumes (MẤT DATA DB!) docker compose -f docker-compose.prod.yml down -v # Khởi động lại 1 service docker compose -f docker-compose.prod.yml restart workforce-web
🔄 Cập nhật code mới
# Sau khi pull code mới từ Git: git pull origin main # Rebuild và restart (downtime ~1-2 phút) docker compose -f docker-compose.prod.yml up -d --build --no-deps workforce-web workforce-api # Hoặc rebuild toàn bộ: docker compose -f docker-compose.prod.yml up -d --build
📋 Xem log & trạng thái
# Xem trạng thái các container docker compose -f docker-compose.prod.yml ps # Xem log backend (50 dòng cuối) docker compose -f docker-compose.prod.yml logs --tail=50 workforce-api # Theo dõi log realtime docker compose -f docker-compose.prod.yml logs -f workforce-api
🗑️ Dọn dẹp
# Xóa image cũ (giải phóng disk) docker image prune -f # Xóa cache build (force rebuild sạch) docker builder prune -f # Xem disk usage của Docker docker system df

Quản lý Database
Migration, backup và kết nối trực tiếp
📦 Backup Database (Production)
# Backup vào file .bak docker exec workforce-db /opt/mssql-tools18/bin/sqlcmd \ -S localhost -U sa -P "$SA_PASSWORD" \ -C -Q "BACKUP DATABASE [WorkForceManagerDb] \ TO DISK='/var/opt/mssql/backup/wfm_$(date +%Y%m%d).bak'" # Copy file backup ra ngoài container docker cp workforce-db:/var/opt/mssql/backup/. ./backup/
🔌 Kết nối DB từ SSMS (Dev)
# Kết nối từ SQL Server Management Studio: Server: (localdb)\mssqllocaldb # Dev local Server: localhost,1433 # Docker production Database: WorkForceManagerDb Auth: SQL Server Authentication Login: sa Password: [giá trị SA_PASSWORD]
🔄 Tạo Migration mới (sau khi sửa Entity)
# Chạy từ thư mục gốc dự án dotnet ef migrations add TenMigration \ --project backend/src/WorkForceManager.Infrastructure \ --startup-project backend/src/WorkForceManager.WebApi # Apply migration vào DB dev dotnet ef database update \ --project backend/src/WorkForceManager.Infrastructure \ --startup-project backend/src/WorkForceManager.WebApi # Production: migration tự chạy khi backend khởi động (Database.MigrateAsync)

Xử lý lỗi thường gặp
Checklist kiểm tra khi gặp sự cố
❌ localhost:5173 — "This site can't be reached"
🔍
Nguyên nhân: Vite dev server chưa chạy.
# Giải pháp: chạy lệnh trong terminal cd frontend && npm run dev
❌ API lỗi 401 Unauthorized liên tục
🔍
Nguyên nhân: Access token hết hạn và refresh token cũng hết hạn.
# Giải pháp: xóa token cũ trong localStorage rồi đăng nhập lại # Mở DevTools (F12) → Application → Local Storage → xóa wf_access_token và wf_refresh_token # Hoặc đăng xuất rồi đăng nhập lại
❌ Docker: container workforce-api liên tục restart
# Xem log để tìm nguyên nhân docker compose -f docker-compose.prod.yml logs workforce-api # Thường do DB chưa sẵn sàng → đợi 30s và thử lại # Hoặc do SA_PASSWORD trong .env không đúng format SQL Server # (phải có chữ hoa + số + ký tự đặc biệt, VD: SaigonSpices_SA_2024!)
❌ Frontend build lỗi "Cannot find module"
# Xóa node_modules và cài lại cd frontend rm -rf node_modules npm install npm run dev
❌ Trang trắng/đen sau khi deploy Docker
# Nguyên nhân: browser cache cũ. Hard refresh: # Windows/Linux: Ctrl + Shift + R # Mac: Cmd + Shift + R # Nếu vẫn lỗi: rebuild image hoàn toàn từ đầu docker compose -f docker-compose.prod.yml down docker builder prune -f docker compose -f docker-compose.prod.yml up -d --build
❌ Port 80 đã bị chiếm (Windows)
# Tìm tiến trình đang dùng port 80 netstat -ano | findstr :80 # Dừng IIS nếu đang chạy iisreset /stop # Hoặc đổi port trong docker-compose.prod.yml # ports: "8080:80" → truy cập http://192.168.20.127:8080
